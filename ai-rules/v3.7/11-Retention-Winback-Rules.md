# 11 - Retention and Winback Rules

Retention work must be privacy-preserving, measurable, and reversible.

- Do not use browsing content, visited domains, or destination IP history for retention decisions.
- Campaign eligibility must be explainable through account, subscription, usage summary, or billing state.
- Any automated offer, coupon, notification, or downgrade flow must define abuse controls and audit fields.
- Feature flags or config keys must be used for rollout and rollback.
- Admin visibility must distinguish user-facing state from internal experiment metadata.
- Completion reports must state analytics, App, Admin, and Backend impact.

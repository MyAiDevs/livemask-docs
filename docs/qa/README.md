# QA 文档入口

## 1. 职责范围

QA 负责测试策略、测试用例、验收证据、跨仓库回归、E2E、性能、兼容性和发布质量门禁。

## 2. 测试前必须确认

- [ ] 当前测试关联 `TASK-XXXX`
- [ ] 任务单中已列出 In Scope / Out of Scope
- [ ] API、配置、事件、状态机契约已冻结或标注 Draft
- [ ] 受影响仓库和角色已明确
- [ ] 回滚路径和失败补偿路径可测试

## 3. 必须覆盖的测试维度

- 正向路径
- 参数校验和错误码
- 重试、超时、取消、恢复
- 幂等、乱序、重复事件
- 旧版本兼容
- 权限和审计
- 回滚或降级

## 4. 完成标准

- [ ] 测试用例覆盖任务单验收标准
- [ ] Contract check 已执行
- [ ] 跨仓库关键路径已回归
- [ ] 失败路径和回滚路径有证据
- [ ] 残余风险写入任务单或风险台账

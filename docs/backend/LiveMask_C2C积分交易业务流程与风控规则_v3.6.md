# LiveMask C2C 积分交易完整业务流程与风控规则 v3.6

## 一、核心业务流程

### 1. 卖家上架流程（Sell Points）
1. 用户进入「我的积分」→ 点击「出售积分」
2. 输入出售数量 + 期望总价 USDT
3. 系统校验：
   - 余额是否充足
   - 是否达到每日上架次数上限
   - 是否触发风控规则
4. 创建 `points_c2c_listings` 记录，状态 = active
5. 积分暂时冻结（从可用余额中扣除，进入 pending）

### 2. 买家购买流程（Buy Points）
1. 用户浏览 C2C 市场列表
2. 选择心仪挂单 → 点击「立即购买」
3. 确认订单 → 调用支付接口（USDT）
4. 支付成功后：
   - 增加买家积分
   - 扣除卖家积分（已冻结部分释放）
   - 平台收取抽成（USDT 进入平台收益）
   - 记录 `points_c2c_trades`
   - 更新 listing 状态为 sold

### 3. 取消上架
- 只有状态为 active 的 listing 可以取消
- 积分解冻回到卖家可用余额

## 二、完整风控规则（后台可配置）

| 风控维度           | 规则描述                              | 默认值 | 可配置 |
|--------------------|---------------------------------------|--------|--------|
| 每日上架次数       | 单个用户每天最多上架次数              | 5次    | 是     |
| 单次上架最大数量   | 单笔最多出售积分数量                  | 10000  | 是     |
| 每日 C2C 交易额    | 单个用户每天 C2C 交易总金额上限       | 500 USDT | 是   |
| KYC 要求           | 上架积分是否需要完成 KYC              | 否     | 是     |
| 异常价格检测       | 价格偏离参考汇率超过 ±30% 自动审核    | 是     | 是     |
| 新用户限制         | 注册不满 7 天用户禁止上架             | 是     | 是     |
| 关联账号检测       | 同一设备/IP 短时间内多次交易自动风控  | 是     | 是     |

## 三、异常处理机制

- 交易过程中支付失败 → 自动回滚积分冻结
- 发现刷单/洗钱行为 → 自动冻结相关 listing + 通知人工审核
- 申诉通道：买卖双方均可对交易发起申诉（复用现有申诉系统）

此流程已与收益模型、订阅支付、质量申诉形成完整闭环。

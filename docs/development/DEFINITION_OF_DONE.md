# Definition of Done

> 不同类型任务的 Done 标准不同。此文档用于避免“代码写完但闭环没完成”。

## 1. 通用 Done

- [ ] 有 `TASK-XXXX`
- [ ] Scope 未扩展
- [ ] 影响仓库已检查
- [ ] 契约文档已更新
- [ ] 测试或手工验证证据已记录
- [ ] 回滚策略已记录
- [ ] 未完成项有后续 TASK

## 2. Backend Done

- [ ] 请求校验、错误码、状态流转完整
- [ ] 幂等、重试、审计已处理
- [ ] 数据库迁移、索引和约束已验证
- [ ] App / NodeAgent 兼容性已检查
- [ ] API contract 或 event contract 已更新

## 3. App Done

- [ ] 成功、失败、取消、重试路径都有用户反馈
- [ ] 本地缓存和旧版本数据兼容
- [ ] Backend 错误码映射到明确行为
- [ ] 支付、登录、连接等关键路径有验证证据
- [ ] 相关 API / config contract 已确认

## 4. NodeAgent Done

- [ ] 配置拉取、版本校验、异常配置处理完整
- [ ] 降级、恢复、重试、上报路径完整
- [ ] Backend API 兼容性已检查
- [ ] 监控、日志、安装或运行故障有排查路径
- [ ] 相关 config / event contract 已更新

## 5. Payment Done

- [ ] Webhook 签名和幂等验证完整
- [ ] 订单状态机和异常补偿完整
- [ ] App 用户反馈路径完整
- [ ] 审计、告警、人工介入入口明确
- [ ] 回滚不会造成权益或资金状态不一致

## 6. Config Hot Reload Done

- [ ] 有默认值
- [ ] 旧版本兼容
- [ ] 非法值处理明确
- [ ] NodeAgent / App 热更新或重启行为明确
- [ ] 回滚和监控指标明确

## 7. Security Done

- [ ] 威胁模型或风险说明已记录
- [ ] Secret 不进入客户端可见配置
- [ ] 权限、审计、告警完整
- [ ] 密钥轮换或回滚策略明确
- [ ] 安全失败路径已验证

## 8. Admin / Frontend Done

- [ ] Admin API contract 已更新或确认无需更新
- [ ] 表单字段、默认值、校验和权限已说明
- [ ] 高风险操作具备审计、确认和必要的复核流程
- [ ] 运营人员的异常处理路径明确
- [ ] 验证截图或接口响应已记录

## 9. Operations / DevOps Done

- [ ] 部署步骤和回滚步骤可执行
- [ ] 环境变量、Secret、证书和迁移影响已检查
- [ ] 监控指标、告警和 Dashboard 已同步
- [ ] 值班、升级和事故响应路径已明确
- [ ] 灾备或回滚验证结果已记录

## 10. QA Done

- [ ] 测试用例覆盖任务单验收标准
- [ ] Contract、E2E、兼容和失败路径已验证
- [ ] 回滚、降级或补偿路径已验证
- [ ] 残余风险写入任务单或风险台账
- [ ] 发布质量门禁结论明确

## 11. Product Done

- [ ] 用户价值、业务指标和验收标准明确
- [ ] In Scope / Out of Scope 明确
- [ ] 灰度、回滚、客服话术和运营动作明确
- [ ] 范围外事项有后续 TASK
- [ ] 发布后观察窗口和复盘入口明确

# Definition of Done

> 不同类型任务的 Done 标准不同。此文档用于避免“代码写完但闭环没完成”。

## 1. 通用 Done

- [ ] 有 `TASK-XXXX`
- [ ] Scope 未扩展
- [ ] 影响仓库已检查
- [ ] 契约文档已更新
- [ ] 测试或手工验证证据已记录
- [ ] 回滚策略已记录
- [ ] 未完成项有后续 TASK

## 2. Backend Done

- [ ] 请求校验、错误码、状态流转完整
- [ ] 幂等、重试、审计已处理
- [ ] 数据库迁移、索引和约束已验证
- [ ] App / NodeAgent 兼容性已检查
- [ ] API contract 或 event contract 已更新

## 3. App Done

- [ ] 成功、失败、取消、重试路径都有用户反馈
- [ ] 本地缓存和旧版本数据兼容
- [ ] Backend 错误码映射到明确行为
- [ ] 支付、登录、连接等关键路径有验证证据
- [ ] 相关 API / config contract 已确认

## 4. NodeAgent Done

- [ ] 配置拉取、版本校验、异常配置处理完整
- [ ] 降级、恢复、重试、上报路径完整
- [ ] Backend API 兼容性已检查
- [ ] 监控、日志、安装或运行故障有排查路径
- [ ] 相关 config / event contract 已更新

## 5. Payment Done

- [ ] Webhook 签名和幂等验证完整
- [ ] 订单状态机和异常补偿完整
- [ ] App 用户反馈路径完整
- [ ] 审计、告警、人工介入入口明确
- [ ] 回滚不会造成权益或资金状态不一致

## 6. Config Hot Reload Done

- [ ] 有默认值
- [ ] 旧版本兼容
- [ ] 非法值处理明确
- [ ] NodeAgent / App 热更新或重启行为明确
- [ ] 回滚和监控指标明确

## 7. Security Done

- [ ] 威胁模型或风险说明已记录
- [ ] Secret 不进入客户端可见配置
- [ ] 权限、审计、告警完整
- [ ] 密钥轮换或回滚策略明确
- [ ] 安全失败路径已验证

# LiveMask 积分经济体系技术白皮书 v3.6（最终版）

**版本**：v3.6  
**日期**：2026-05-11  
**状态**：生产就绪

## 1. 背景与目标

LiveMask 采用 **USDT + 积分** 双轨收益体系，目标是：

- 让赞助商（Sponsor）和推广大使获得除 USDT 之外的额外收益形式
- 增加普通用户粘性（购买套餐赠送积分 + 积分可购买套餐）
- 构建平台内部经济闭环（C2C 交易 + 平台抽成）
- 通过后台可配置规则实现灵活的商业策略调整

## 2. 核心设计原则

1. **后台完全可配置**：所有 earning 规则、风控参数、平台抽成比例均通过 `system_configs` 管理，无需改代码。
2. **双轨分离**：USDT 用于真实收益结算，积分用于平台内循环。
3. **风控优先**：C2C 交易必须经过价格偏离检测、每日限额、KYC 等多层风控。
4. **可追溯**：积分 earning 和消费均支持通过申诉系统进行追溯调整。
5. **数据驱动**：通过每日定时任务统计发行量、交易量、通胀率等关键指标。

## 3. 核心数据模型

- `points_balances`：用户/赞助人/推广大使积分余额
- `points_transactions`：积分流水（earning / spend / transfer / c2c_trade）
- `points_c2c_listings`：C2C 挂单
- `points_c2c_trades`：C2C 成交记录（含平台抽成）

## 4. 关键业务流程

### 4.1 积分 Earning
- 赞助节点贡献流量/质量 → 按 `node_points_rate` 获得积分
- 用户购买套餐 → 按 `plan_purchase_bonus_rate` 赠送积分
- 推广大使被邀请用户消费 → 按 `promoter_points_rate` 获得积分

### 4.2 积分消费
支持混合支付（积分 + USDT），优先使用积分。

### 4.3 C2C 积分交易
1. 卖家上架（数量 + 单价）
2. 系统风控校验（价格偏离、每日限额）
3. 买家使用 USDT 购买
4. 平台抽成（默认 8%）
5. 积分与 USDT 原子性划转

## 5. 风控体系

- 价格偏离检测（市场价 ±阈值）
- 每日交易限额（按 KYC 等级区分）
- 大额交易强制 KYC
- 异常行为熔断（短时间内高频交易自动冻结）
- 与现有申诉系统深度联动

## 6. 后台配置能力

所有核心参数均可通过 Admin 后台「积分经济配置」页面实时调整，并支持影响模拟预览。

## 7. 技术架构

- Go 后端三层架构（Handler / Service / Repository）
- Redis 缓存热点配置与余额
- 定时任务（每日统计、过期处理、风控重置）
- 与现有订阅、收益、申诉、通知系统深度集成

## 8. 未来扩展方向

- 积分可用于更多场景（节点加速、专属节点、活动报名等）
- 积分等级体系（不同等级享受不同 earning 加成）
- 积分借贷 / 质押功能（进阶版本）

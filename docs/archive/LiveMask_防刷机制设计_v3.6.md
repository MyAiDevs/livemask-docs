# LiveMask 防刷机制设计 v3.6（最终闭环版）

## 1. 整体防刷策略
多层防御：客户端本地黑白名单 + 后端实时风控 + 威胁狩猎 + 人工申诉复核

## 2. 威胁狩猎（Threat Hunting）优化（v3.6 重大更新）
- 命中后**不直接处罚**，而是进入 `quarantine`（隔离观察期 7 天）
- **拦截效果**：quarantine 期间，该节点对所有付费用户不可见（客户端拉取节点列表时自动过滤）
- 同时自动在 `appeals` 表创建 `source = 'system_hunting'` 的系统申诉记录
- 管理员可在后台进行人工复核，7天内未处理则自动执行原处罚
- 此设计大幅降低误判对大使/节点造成的伤害

## 3. 黑白名单联动
- 威胁情报聚合服务自动将高置信度情报写入黑名单
- 客户端本地缓存 + 版本哈希校验 + 自动回滚
- 用户举报 → 进入申诉系统 → 人工确认后加入黑名单

## 4. 推广大使防刷
- 异常邀请行为直接影响 Tier 和忠诚度加成
- 多次违规可直接降低 Tier 或暂停加成资格

## 5. C2C 防刷
- 高频低价交易、关联账号交易等行为自动标记
- 配合 C2C 平台补贴机制，异常交易不享受补贴

# LiveMask 高级可观测性（SLO + Tracing）设计 v3.6

**创建日期**：2026-05-10  
**优先级**：P0（运维能力关键）

## 1. 核心 SLO 定义（建议值）

| 服务/功能             | SLO 目标          | 错误预算（每月） | 关键指标                     |
|-----------------------|-------------------|------------------|------------------------------|
| API 可用性            | 99.9%             | 43.2 分钟        | 5xx 错误率 + 延迟 P99        |
| NodeAgent 连接成功率  | 99.5%             | 3.6 小时         | 首次连接成功率 + 心跳成功率  |
| 支付成功率            | 99.8%             | 1.44 小时        | 支付回调成功率               |
| 收益计算任务及时率    | 99.0%             | 7.2 小时         | 每日 02:00 前完成            |
| WebSocket 实时服务    | 99.5%             | 3.6 小时         | 连接保持率 + 消息投递成功率  |

## 2. 技术栈推荐
- **Metrics + Tracing**：OpenTelemetry + Jaeger + Prometheus
- **日志**：Loki + Grafana
- **告警**：Alertmanager + PagerDuty / 企业微信
- **可视化**：Grafana + 自建 Dashboard

## 3. 关键实现点
- 所有后端服务统一接入 OpenTelemetry SDK
- NodeAgent 心跳 + 配置热更新链路 tracing
- 收益计算、申诉处理等异步任务 tracing
- Error Budget 仪表盘（每月自动重置）

## 4. 告警分级
- **P0**：核心服务不可用、支付失败率 > 1%、收益计算任务失败
- **P1**：NodeAgent 大量 degraded、SLO 接近耗尽
- **P2**：非核心功能异常、性能轻微下降
